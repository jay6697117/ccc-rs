# Phase 4: ccc-mcp — Model Context Protocol 协议层

## 目标
实现 MCP 协议核心，支持动态扩展 AI 的上下文和工具集。

## 核心功能
1. **JSON-RPC 核心**：
   - 实现消息序列化/反序列化。
   - 请求、响应与通知的处理逻辑。
2. **传输层抽象 (Transport Layer)**：
   - `StdioTransport`：通过标准输入输出与外部进程通信。
   - `HttpTransport` (可选/后续)：通过 HTTP 调用远程服务器。
3. **客户端实现**：
   - 初始化握手 (initialize)。
   - 列表查询 (List Tools/Resources)。
   - 调用执行 (Call Tool)。
4. **进程管理**：
   - 自动拉起、监控及优雅重启 MCP 服务器进程。

## 依赖项
- `tokio` (进程交互)
- `ccc-core` (核心消息模型)
