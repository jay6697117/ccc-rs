# Phase 3: ccc-api — 流式接口与网络层

## 目标
构建高性能、支持多厂商（Claude/OpenAI/GCP）的流式 API 客户端。

## 核心功能
1. **流式 HTTP 客户端**：
   - 基于 `reqwest` 的异步非阻塞请求。
   - 支持 HTTP/2 和连接池。
2. **SSE (Server-Sent Events) 解析器**：
   - 实时解析流式响应块。
   - 错误边界处理与流恢复。
3. **重试逻辑 (Retry Strategy)**：
   - 指数退避重试。
   - 针对速率限制 (429) 和临时网络错误 (503) 的优化。
4. **多提供者适配**：
   - 抽象通用 Request/Response Trait。

## 依赖项
- `ccc-auth` (令牌注入)
- `tokio-stream` (异步流处理)
- `serde_json` (数据解析)
